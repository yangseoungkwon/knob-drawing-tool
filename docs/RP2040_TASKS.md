# RP2040 상세 태스크

## 1) 목표와 범위

- RP2040 보드(예: Raspberry Pi Pico)를 USB HID 마우스로 동작시킨다.
- 좌/우 노브 회전을 각각 Y/X 커서 이동으로 매핑한다.
- 좌/우 노브 버튼 입력을 클릭 이벤트로 매핑한다.
- 브라우저의 `pages/knob-drawing.html`에서 실사용 가능한 입력 품질(지연, 오입력)을 확보한다.

## 2) 구현 기준(Definition of Done)

- USB 연결 시 OS에서 표준 마우스로 인식된다.
- 회전 1스텝당 커서 이동량이 일정하고 방향이 직관적이다.
- 버튼 디바운스가 적용되어 클릭 중복 발생이 없다.
- 10분 연속 테스트에서 입력 멈춤/튐 현상이 재현되지 않는다.
- 문서(`핀맵`, `빌드 방법`, `트러블슈팅`)가 정리되어 다른 사람이 재현 가능하다.

## 3) 기술 선택

- 권장: Arduino-Pico 코어 + `Mouse`(TinyUSB 기반) 라이브러리
- 대안: Pico SDK(C/C++) + TinyUSB 직접 구성

초기 속도를 위해 1차는 Arduino-Pico 경로로 진행하고, 성능 이슈가 있으면 Pico SDK로 전환한다.

## 4) 하드웨어 태스크

### T1. 부품/배선 확정

- [ ] 노브 2개(EC11 등), 푸시 스위치 포함 모델 확정
- [ ] RP2040 GPIO 핀맵 작성(A/B/SW, GND, 3.3V)
- [ ] 풀업/풀다운 전략 확정(내장 Pull-up 우선)
- [ ] 브레드보드 배선 후 쇼트/역결선 점검

### T2. 신호 안정화

- [ ] 엔코더 A/B 채널 노이즈 확인
- [ ] 버튼 디바운스(소프트웨어 10~20ms) 적용
- [ ] 공통 GND 접속 상태 점검

## 5) 펌웨어 태스크

### T3. 프로젝트 골격

- [ ] 보드 패키지 설치(Arduino IDE 또는 PlatformIO)
- [ ] 기본 스케치 생성 및 시리얼 로깅 확인
- [ ] USB HID 마우스 초기화 코드 추가

### T4. 엔코더 입력 처리

- [ ] 좌/우 노브 각각 상태 머신 또는 인터럽트 기반 디코딩 구현
- [ ] 회전 방향(CW/CCW) 판별 로직 검증
- [ ] 스텝 누적 -> 커서 델타 변환 함수 구현

### T5. 버튼 처리

- [ ] 좌 노브 버튼 -> 좌클릭(토글/홀드 중 선택) 구현
- [ ] 우 노브 버튼 -> 우클릭 또는 모드 전환 신호 구현
- [ ] 동시 클릭(좌+우) 조합 입력 판별 구현

### T6. HID 출력

- [ ] X/Y 이동량 전송(`Mouse.move`) 주기 제어
- [ ] 클릭 이벤트 전송(`press`/`release` 또는 `click`) 구현
- [ ] 과도 이동 방지(최대 속도/가속 제한) 적용

## 6) 웹 연동 태스크

### T7. 브라우저 동작 검증

- [ ] `pages/knob-drawing.html`에서 커서 이동/클릭 동작 확인
- [ ] 캔버스 가장자리에서 동작 안정성 확인
- [ ] 이동 모드/그리기 모드 전환이 의도대로 동작하는지 확인

### T8. UX 튜닝

- [ ] 스텝당 픽셀 이동량 기본값 확정
- [ ] 느린 회전/빠른 회전 감도 커브 조정
- [ ] 우클릭 기능(지우개/색상 전환) 우선 정책 확정

## 7) 테스트 태스크

### T9. 기능 테스트

- [ ] 방향 정확도 테스트(상/하/좌/우)
- [ ] 연속 회전 스트레스 테스트(5분 이상)
- [ ] 버튼 연타/동시 클릭 테스트

### T10. 회귀 테스트

- [ ] USB 재연결 후 즉시 동작 확인
- [ ] PC 재부팅 후 재인식 확인
- [ ] 브라우저 탭 전환/포커스 변경 시 오동작 여부 확인

## 8) 문서화 태스크

- [ ] 핀맵 다이어그램 또는 표 추가
- [ ] 펌웨어 빌드/업로드 절차 정리
- [ ] 자주 발생하는 이슈와 해결법(방향 반전, 튐, 미인식) 정리

## 9) 바로 시작할 첫 스프린트(추천)

1. T1(핀맵 확정) + T3(프로젝트 골격)
2. T4(엔코더 입력) 최소 구현
3. T6(HID 이동만 먼저) 연결
4. 브라우저에서 커서 이동 확인 후 T5 버튼 입력 추가

## 10) 리스크와 대응

- 엔코더 품질 편차: 소프트웨어 필터 + 샘플링 주기 조정
- USB HID 호환성: OS별 테스트(macOS/Windows)
- 감도 튜닝 장기화: 런타임 파라미터(시리얼 명령) 도입 검토
